# Forensic Synthesis Guide — Doc 01 Narrative Architecture

> Reference for building forensic causation reports. Load when synthesizing executive summaries, conclusions, or key arguments.

---

## Document Structure

Doc 01 (Forensic Causation Report) follows this narrative flow:

### 1. Executive Summary (`forensic_findings.executive_summary`)
- 2-3 paragraphs, clinical tone
- Opens with property identification and inspection context
- States key finding: storm caused damage exceeding carrier scope
- Quantifies: "X storm events confirmed by NOAA within Y miles"
- References HAAG standards, building codes, manufacturer specs

### 2. Damage Summary (`forensic_findings.damage_summary`)
- Component-by-component walkthrough
- Each trade gets a subsection (roofing, gutters, siding, interior)
- Clinical language: "exhibits," "demonstrates," "consistent with"
- Never uses "we believe" or "in our opinion" — state findings as facts with citations

### 3. Evidence Cascade (logical flow)
Build the argument in this order — each layer corroborates the next:

1. **Ground-level soft metals** (gutters, downspouts, window wraps, AC units)
   - Lowest threshold (0.75") — if dented, hail was at least this size
   - Cannot be fabricated — permanent deformation in exposed horizontal surfaces

2. **Soft metal siding** (aluminum siding, trim, fascia)
   - Directional impact patterns corroborate wind-driven hail
   - Impact density increases on windward elevations

3. **Roof system** (shingles, underlayment, flashing)
   - Mat fracture, granule displacement, bruising
   - Reference HAAG functional damage threshold (1.0" for new, 0.75" for aged)
   - Exposure measurement = age indicator (5" = pre-metric = unrepairable)

4. **Interior damage** (if present)
   - Water infiltration = functional failure of weather barrier
   - Traces causation chain from exterior breach to interior effect

### 4. Key Arguments (`forensic_findings.key_arguments`)
Array of objects: `{title, content}`

Common argument categories:
- **HAAG threshold exceeded** — NOAA-confirmed hail size vs material threshold
- **Code violations** — carrier scope omits code-required components (I&W, drip edge, underlayment)
- **Inspector contradictions** — carrier inspector findings vs physical evidence
- **Matching doctrine** — NAIC MDL-902, "like kind and quality" requirements
- **Shingle exposure incompatibility** — 5" exposure = unrepairable with current metric products
- **Per-shingle vs per-square** — carrier counted per-shingle hits instead of per-SQ industry standard

### 5. Conclusion (`forensic_findings.conclusion_paragraphs`)
- Restate findings with authority
- Reference total USARM scope value vs carrier scope
- List specific omissions
- Close with professional recommendation for full replacement

## Language Standards

### Clinical Tone (All Roles)
- "The inspection revealed..." not "We found..."
- "Physical evidence demonstrates..." not "We believe..."
- "Consistent with hail impact per HAAG Engineering standards" not "Looks like hail damage"
- Cite standards by number: RCNYS R905.1.1, ASTM D7158, HAAG Residential Roof Inspector certification

### Contractor Mode (UPPA Compliance)
- Frame as "code-compliance documentation" not "supplement appeal"
- "This report documents observed conditions" not "We appeal the carrier's denial"
- Never: "on behalf of," "demand," "appeal," "supplement and appeal"
- Never cite: 11 NYCRR 216, Insurance Law § 2601

### PA/Attorney Mode
- Full advocacy language permitted
- "We respectfully demand revised consideration"
- Can cite regulatory statutes and bad faith precedent

## Synthesis Prompts — Key Principles

1. **Never hallucinate measurements** — only use values from EagleView/HOVER extraction
2. **Never hallucinate weather data** — only cite NOAA-confirmed events or HailTrace reports
3. **Every claim gets the exposure check** — measure butt-to-butt in photos, flag 5" as unrepairable
4. **Carrier playbook informs strategy** — load before synthesis, reference proven arguments
5. **Evidence map drives narrative** — each disputed item must have photo, measurement, code, or weather backing
